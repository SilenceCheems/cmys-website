# CMYS 人生模拟器 — 优化设计规格

> 基于 2026-06-27 新增需求，覆盖难度、交互、事件重复、评分四个维度。

---

## 一、难度系统：分层致死网络

### 1.1 分层难度曲线

| 年龄段 | 即死事件数 | 致死属性阈值 | 随机意外/年 | 设计意图 |
|--------|-----------|-------------|-----------|---------|
| 少年 6-17 | 2~3 个 | 归零才死 | 无 | 明显作死型，教育玩家熟悉机制 |
| 青年 18-30 | 5~7 个 | 归零才死 | 1% | 冒险/社会陷阱/健康赌博 |
| 壮年 31-60 | 8~12 个 | ≤10 即死 | 2% | 核心绞肉机，大部分死亡发生在此 |
| 晚年 61-100 | 5~7 个 | ≤10 即死 | 3% | 身体脆弱，小病致命 |

全游戏即死事件总计约 20~29 个。

### 1.2 三层致死类型

| 层级 | 机制 | 说明 |
|------|------|------|
| 即死选项 | 事件中某个选项 `isLethal: true`，选错即死 | 前端加明显警示（红色标记/文案暗示），但不禁止 |
| 属性陷阱 | 选项扣除大量致命属性（-15~-40） | 隐蔽但非即死，把玩家推到危险线 |
| 随机意外 | `advanceYears` 循环中按概率触发 | 不依赖事件，模拟车祸/急病/天灾 |

### 1.3 致死阈值分龄

- 少年/青年期：致命属性归零（`≤0`）才死
- 壮年/晚年期：致命属性降到 `≤10` 即死

`checkDeath` 函数新增 `lethalThreshold` 参数，根据年龄动态切换。

### 1.4 体质衰减（维持现状）

30 岁起每 5 年 -1，仅在各整 5 年节点触发。

## 二、事件系统：去重与扩充

### 2.1 全局 maxTriggers 覆盖

| 事件类型 | maxTriggers | 说明 |
|---------|------------|------|
| 高强度/特殊事件 | 1 | 一生一次（如高考、初恋、婚礼） |
| 日常事件 | 2 | 可复现但不过分 |
| 通用 filler | 3 | 预留冗余 |

所有参数化事件补齐 `maxTriggers`，当前缺失的事件补全。

### 2.2 冷却期机制

`EventBase` 新增 `cooldownYears` 字段：
- 日常事件：8~12 年
- 高强度事件：15~20 年

实现：`triggeredEventIds` 改为 `Map<string, number>`，记录事件 ID → 最后触发年龄。`isEventEligible` 中检查 `age - lastTriggeredAge >= cooldownYears`。

### 2.3 事件池扩充

| 类型 | 当前 | 目标 | 增量 |
|------|------|------|------|
| 锚点事件 | 22 | 28~30 | +6~8 |
| 参数化事件 | 68 | 100~110 | +32~42 |
| **总计** | **90** | **128~140** | **+38~50** |

新增事件优先填充青年期和壮年期（交互最密集的年龄段）。

所有事件标题保持 CMYS 四字首字母规范（用 `pypinyin` 验证 `initials == 'cmys'`）。

## 三、交互系统：Reigns 式卡片

### 3.1 复用运势卡视觉

事件的 Reigns 卡片复用 `FortuneSystem` 的 CardPhase 视觉语言：
- 深色底（`bg-[#121212]`）
- 六边形 clipPath（`40px` 斜切角）
- 1px 网格线、四角装饰线
- 中心径向渐变辉光
- 排版：`font-mono` 标签 + `font-serif` 标题 + 分隔线 + 正文

### 3.2 交互行为

**鼠标拖拽：**
- `drag="x"` 限制水平拖拽
- 拖动时卡片跟随旋转 ±15°（`rotate: dragX * 0.1`）
- 松手超过屏幕宽度 30% → 确认选择
- 未超过 → spring 弹回原位

**键盘：**
- `←` 选择左选项，`→` 选择右选项
- 按键时触发飞出动画（`x: ±300, opacity: 0`）

**移动端：**
- 左滑 / 右滑手势（Motion 的 `drag` 原生支持触摸）

### 3.3 视觉反馈

- 左拖/左键：背景渐变冷色（深蓝/墨绿）
- 右拖/右键：背景渐变暖色（琥珀/暗金）
- 阈值线：半透明竖线提示临界点
- 拖动时卡片边缘出现方向辉光
- 选项文字在拖动达到阈值时高亮

### 3.4 组件改造

- `LifeEventCard.tsx` → 替换为 `ReignsCard`，内部集成 drag + keyboard
- `LifeYouthStage` / `LifeMidlifeStage` / `LifeElderStage` 中的事件渲染统一使用新卡片
- 保留 `resultText` 叙事结果展示（`LifeEventResult` 或内嵌在卡片翻面动画中）

## 四、评分系统：三层架构

### 4.1 公式

```
总分 = 基础分（35%） + 成就分（40%） + 叙事分（25%）
满分 1000
```

### 4.2 基础分（0~350）

```
基础分 = (六维属性和 / 600) × 350
```

### 4.3 成就分（0~400）

预设 15~20 个隐藏成就，每个 20~50 分。

**示例成就：**

| 成就 | 条件 | 分值 |
|------|------|------|
| 大起大落 | 财富最高-最低 ≥70 | 40 |
| 向死而生 | 逃过一次即死判定（属性跌到 ≤5 但未死） | 50 |
| 伯牙子期 | 知己好感达 100 | 30 |
| 逆天改命 | 运势曾归零后又恢复到 80+ | 40 |
| 少年白头 | 30 岁前体质曾跌破 10 | 30 |
| 情圣 | 触发所有爱情相关事件 | 30 |
| 不死鸟 | 逃过 3 次以上即死选项 | 50 |
| 白手起家 | 家境曾跌破 10 后又恢复到 80+ | 40 |
| 学富五车 | 智力从未跌破 50，且触发过所有学业事件 | 30 |
| 百年孤独 | 活到 100 岁 | 50 |
| 早夭 | 20 岁前死亡 | 20（安慰奖） |
| 杀手本能 | 某属性因你的选择导致另一个 NPC 关系破裂 | 20 |

结算时展示已触发成就列表，未触发的灰掉显示（激励重玩）。

### 4.4 叙事分（0~250）

```
叙事分 = 事件多样性分 + 关系深度分 + 职业巅峰分 + 隐藏结局加权
```

| 子项 | 公式 | 上限 |
|------|------|------|
| 事件多样性 | 不同事件数 × 4 | 100 |
| 关系深度 | 知己好感绝对值 | 50 |
| 职业巅峰 | 职业最高等级 × 8 | 80 |
| 隐藏结局 | 特殊条件触发额外加分 | 20 |

### 4.5 评星阈值

| 星级 | 分数 | 预期占比 |
|------|------|---------|
| 1★ | < 150 | ~20% |
| 2★ | 150-350 | ~30% |
| 3★ | 350-550 | ~25% |
| 4★ | 550-750 | ~15% |
| 5★ | 750+ | ~10% |

（占比数据待实际测试后校准）

### 4.6 结局称号

沿用现有五档称号体系（四字短语→成语→五言→骈文），每档 3~5 个随机选取。新增成就触发的专属称号（如触发"不死鸟"成就 → "不死鸟"专属称号）。

## 五、技术要点

### 5.1 引擎改动

| 文件 | 改动 |
|------|------|
| `engine/types.ts` | `EventBase` 新增 `cooldownYears?: number`；`triggeredEventIds` 从 `Set<string>` 改为 `Map<string, number>`（id → 最后触发年龄） |
| `engine/death.ts` | `checkDeath` 新增 `lethalThreshold` 参数，分龄切换；新增 `checkRandomDeath(age)` 按概率返回随机意外 |
| `engine/events.ts` | `isEventEligible` 新增冷却期检查；`selectEvent` 排除冷却中事件 |
| `engine/reducer.ts` | `advanceYears` 循环中调用 `checkRandomDeath`；`RESOLVE_EVENT` 后记录 `triggeredAge` |
| `engine/ending.ts` | 重构为三层架构；新增成就枚举和判定逻辑 |

### 5.2 组件改动

| 文件 | 改动 |
|------|------|
| `components/LifeEventCard.tsx` | 替换为 Reigns 式拖拽卡片，集成 drag + keyboard |
| `components/LifeDeathScreen.tsx` | 新增成就展示列表（已触发/未触发） |
| 各 Stage 组件 | 事件渲染统一使用新 `ReignsCard`，移除旧的按钮式选择 |

### 5.3 数据改动

| 文件 | 改动 |
|------|------|
| `data/life/events-parametric.ts` | 补齐 `maxTriggers` + `cooldownYears`；新增 32~42 个事件 |
| `data/life/events-anchors.ts` | 新增 6~8 个锚点事件 |
| `data/life/endings.ts` | 新增成就列表数据 |
| `data/life/talents.ts` | 视需要微调天赋平衡 |

### 5.4 新增文件

```
src/
  engine/
    achievements.ts       # 成就判定逻辑（纯函数）
  data/life/
    achievements.ts       # 成就数据和条件配置
```
